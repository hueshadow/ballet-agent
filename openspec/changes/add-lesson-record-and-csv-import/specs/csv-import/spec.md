## ADDED Requirements

### Requirement: CSV 批量导入历史课时
系统 SHALL 支持用户通过 CSV 文件批量导入历史上课记录。流程：下载模板 → 电脑端填写 → 上传 CSV → 预览校验 → 确认写入。仅支持 UTF-8 编码的 .csv 文件，单批最多 1000 条。

#### Scenario: 下载 CSV 模板
- **WHEN** 用户在"我的" → 导入数据页点击"下载模板"
- **THEN** 下载一个 CSV 模板文件，包含字段：日期、机构、老师、课型、级别、时长、笔记、身体评分、老师评分、标签、愉悦度

#### Scenario: 上传并预览
- **WHEN** 用户选择已填写的 CSV 文件上传
- **THEN** 前端解析文件，展示预览页（前 N 条数据 + 总记录数 + 校验结果）

#### Scenario: 校验必填字段
- **WHEN** CSV 中某行缺少必填字段（日期/机构/老师/课型/时长）
- **THEN** 该行标红，显示"缺少必填字段 XXX"

#### Scenario: 日期格式校验
- **WHEN** CSV 中日期不符合 YYYY-MM-DD 格式
- **THEN** 该行标红，提示"日期格式错误"

#### Scenario: 疑似重复检测
- **WHEN** 同一天 + 同机构 + 同老师 + 同时长的记录已存在
- **THEN** 该行标黄，提示"疑似重复"，用户可选择跳过或强制导入

#### Scenario: 自动新建不存在的机构/老师
- **WHEN** CSV 中的机构名或老师名在系统中不存在
- **THEN** 自动创建对应的机构/老师记录，标记为"待补完信息"

#### Scenario: 批量写入与进度
- **WHEN** 用户确认导入
- **THEN** 云函数分批写入（每批 50-100 条），界面显示进度条
- **THEN** 全部写入完成后跳转记录列表，所有导入记录的来源标记为 csv_import

#### Scenario: 超出上限
- **WHEN** CSV 文件超过 1000 条
- **THEN** 提示"单次最多导入 1000 条，请拆分文件后分批导入"
